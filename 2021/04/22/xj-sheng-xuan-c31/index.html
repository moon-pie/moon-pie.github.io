A.Path

Description

给定一张有向图 G=(V,E)G = (V,E)G=(V,E)，V={1,2,…,n}V = \{1,2,\dots,n\}V={1,2,…,n}。每个点有个标记 fif_ifi​，为 LLL 或 RRR 或 BBB 。

(i,j)∈E⟺(fi=L∧j<i)∨(fi=R∧j>i)∨(fi=B)(i,j)\in E \Longleftrightarrow (f_i = L \wedge j<i) \vee (f_i = R \wedge j>i) \vee (f_i = B) (i,j)∈E⟺(fi​=L∧j<i)∨(fi​=R∧j>i)∨(fi​=B)

Solution

part1

暴力状压DP，考虑DP状态 f[S][u]f[S][u]f[S][u] 表示经过状态为 SSS 的点集，停留在 uuu 点的方案数。

时间复杂度 O(n×2n)\mathcal O(n\times 2^n)O(n×2n)。期望得分：50 pts。

正解

只考虑前 iii 个节点，他们在最终的路径状态中一定是若干条链。由此我们把 DP 状态设为 f[i][j]f[i][j]f[i][j] 表示前 iii 个点构成 jjj 条链的方案数。

新加进来分情况转移：

  • 第 iii 个点是 ′L′'L'′L′ 或 ′B′'B'′B′：1、加在一条非1开头链的head；2、连接两条链。
  • 第 iii 个点是 ′R′'R'′R′ 或 ′B′'B'′B′：1、连在前面某一条链的tail；2、新开一条链。

时间复杂度 O(n2)\mathcal O(n^2)O(n2)，数据范围好像还能开大一点。

B. Unknow

Description

给你一棵树，树上每个节点用 ki,bi,li,rik_i,b_i,l_i,r_iki​,bi​,li​,ri​ 表示线段 y=kix+bi  (li≤x≤ri)y = k_ix+b_i\;(l_i\le x\le r_i)y=ki​x+bi​(li​≤x≤ri​)。

每次询问给出 u,v,wu,v,wu,v,w，问从 uuu 到 vvv 这条链上所有线段与直线 x=wx = wx=w 的交点的最大纵坐标。

Solution

part1

如果只是在一个序列上，可以用李超线段树轻松维护。

考虑树剖，于是线段树套李超树的 O(nlog4n)\mathcal O(n\mathrm{log^4}n)O(nlog4n) 的算法横空出世，并且获得和暴力老哥同分的好成绩。

期望得分：13 pts。

李超线段树不具有像普通线段树一样的差分性，这就意味这：李超线段树不能访问历史版本（可持久化）。

考虑把所有询问离线，不难发现：对于一条树上的路径，除了最后的一段外，其他所有都是关于这条重链的前缀。

询问是前缀，加上可以离线，我们就只需要往李超线段树里加线段，然后查询就可以了。这部分就可以做到 O(nlog⁡2n)\mathcal{O(n\log^2 n)}O(nlog2n) 的复杂度。

难点在于，最后 O(n)O(n)O(n) 那段不是前缀的如何处理。

part2

O(nlog⁡3n)\mathcal{O(n\log^3 n)}O(nlog3n) 的做法是，暴力分治。

期望得分：37～100 pts。

正解

询问一段区间，我们想到用线段树。怎么用？

首先，线段树每一个维护一棵李超线段树，dfn\mathrm{dfn}dfn 中一段区间内点所代表的线段的信息。

然后，类似线段树分治的做法，我们将所有的区间询问分到线段树上的 log⁡n\log nlogn 段，并把询问存到线段树的节点中。

最后，我们只要递归建树，父亲合并左右儿子时做一次李超线段树合并就可以了。

Q&A

  • 李超线段树如何合并？

大体参考线段树合并，不同之处在于合并相同位值的点时做一次李超线段树的 insert\mathrm{insert}insert 的操作。

  • 时间复杂度？

按上面所讲，李超线段树合并部分的复杂度为 O(nlog⁡2n)\mathcal{O(n\log^2 n)}O(nlog2n)。原2021省选训练45 来的 O(n)O(n)O(n) 个区间询问到线段树上变成有 O(nlog⁡n)O(n\log n)O(nlogn) 个区间，每个区间对应一次李超线段树的查询操作，所以这部分复杂度也是 O(nlog⁡2n)\mathcal{O(n\log^2 n)}O(nlog2n)。

所以总时间复杂度为 O(nlog⁡2n)\mathcal{O(n\log^2 n)}O(nlog2n)，可以通过本题。
