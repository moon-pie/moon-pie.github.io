A. Advertisement Matching

题意

有 nnn 个广告商，第 iii 个有 aia_iai​ 份广告，有 mmm 个人，第 iii 个人最多看 bib_ibi​ 份不同的广告。qqq 次修改，每次将 ax±1a_x\pm 1ax​±1 或者 bx±1b_x\pm 1bx​±1，修改是累计的。问每次修改后是否能将所有广告发完。（保证所有 ai,bia_i,b_iai​,bi​ 在任意时刻为非负整数）

1≤n,m,q≤250000,0≤ai,bj≤2500001\le n,m,q\le 250000,0\le a_i,b_j\le 250000 1≤n,m,q≤250000,0≤ai​,bj​≤250000

结论

对于从大到小排完序的 aaa，需要满足一下条件才是可以的：

∀k∈[1,n]:∑i=1mmin⁡(bi,k)≥∑i=1kai\forall k\in [1,n]:\sum_{i=1}^m{\min(b_i,k)}\ge \sum_{i=1}^k a_i ∀k∈[1,n]:i=1∑m​min(bi​,k)≥i=1∑k​ai​

构造证明（口胡）：

略。

网络流严谨证明:

考虑根据题意建出网络流模型：从源点 SSS 向 nnn 个点连边，流量为 aia_iai​ 表示第 iii 个广告。每个广告分别向另外 mmm 个点连一条流量为 111 的边。然后让这 mmm 个点向汇点 EEE 连 mmm 条流量为 bib_ibi​ 的边。显然，该图的最大流如果为 ∑ai\sum a_i∑ai​ 输出 111。（下图 n×mn\times mn×m 条流量为 111 的边没有画出来）

根据最大流 === 最小割，我们来分析一下这个图的最小割是怎么样的。

如上图，设 ∣∁US∣=k\left|\complement _{U}S\right| = k∣∣∣​∁U​S∣∣∣​=k，如果 aia_iai​ 按照从大到小的顺序排序，最小割一定是：

∑i=k+1nai+∑j=1∣T∣bj+∑j=∣T∣+1mk\sum_{i=k+1}^{n}a_i + \sum_{j=1}^{|T|}b_j + \sum_{j = |T|+1}^m{k} i=k+1∑n​ai​+j=1∑∣T∣​bj​+j=∣T∣+1∑m​k

因为要最小，所以，

MaxFlow=MinCut=∑i=k+1nai+∑j=1mmin⁡(bj,k)\mathrm{MaxFlow} = \mathrm{MinCut} =\sum_{i=k+1}^{n}a_i + \sum_{j=1}^{m}\min(b_j,k) MaxFlow=MinCut=i=k+1∑n​ai​+j=1∑m​min(bj​,k)

又因为最大流要大于等于 ∑ai\displaystyle\sum a_i∑ai​，所以，

∑i=k+1nai+∑j=1mmin⁡(bj,k)≥∑i=1nai∑j=1mmin⁡(bj,k)≥∑i=1kai\begin{aligned} \sum_{i=k+1}^{n}a_i + \sum_{j=1}^{m}\min(b_j,k) &\ge \sum_{i=1}^{n}a_i\\ \sum_{j=1}^{m}\min(b_j,k) &\ge \sum_{i=1}^ka_i \end{aligned} i=k+1∑n​ai​+j=1∑m​min(bj​,k)j=1∑m​min(bj​,k)​≥i=1∑n​ai​≥i=1∑k​ai​​

由于每次修改大小为 111 ，这个东西随便线段树维护一下就好了。

D. Just Meeting

题意

给你一张 nnn 个点，mmm 条边的无向图，带边权，不保证联通，需要你将它补成一张完全图（原来不存在的边由你钦定边权，原本存在的边不能改动），边权范围 [1,107][1,10^7][1,107] 。这张图需要满足：对于任意三元环，不能存在一条边严格小于另外两条边。最小化边权之和。

3≤n≤3×105,0≤m≤3×1053\le n\le 3\times 10^5,0\le m\le 3\times 10^5 3≤n≤3×105,0≤m≤3×105

题解

step0

题意转化为：把一个给出的图补成一个完全图，满足任意一个三元环没有严格最小的边权。

step1

连通块之间的边全部连 111，显然最优且合法。所以只要考虑一个联通块内的就行了。

step2

考虑一个连通块是一棵树的情况。

深度差为 222 的两个点，它们两点之间的连边一定是两条已经有的边的最小值。然后可以拓展到任意深度差的两点之间的连边，就是树上路径的最小边权。

step3

考虑图如果不是一棵树的情况。如果我们建一棵最大生成树出来，就可以直接按照 step2\mathrm{step2}step2 里的做就好了。证明：

先考虑一下最大生成树的性质：非树边一定小于等于树上路径中的最小边权。

如果原有的边比 Min_edge(u,v)\mathrm{Min\_edge}(u,v)Min_edge(u,v) 还要小，势必会出现一个有严格最小边权的三元环，分讨即可。

然后直接并查集做就完了。

G. LCS 8

Description

给你一个长度为 NNN 的字符串 SSS，和一个 KKK，请你求出长度为 NNN 的字符串 TTT，满足 LCS(S,T)≥N−K\mathrm{LCS}(S,T) \ge N-KLCS(S,T)≥N−K 的方案数。

1≤N≤50000,0≤K≤31\le N \le 50000,0\le K\le 3 1≤N≤50000,0≤K≤3

Solution

dp套dp。先写一下 LCS\mathrm{LCS}LCS 的 dp 方程：

dpi,j=max(dpi,j−1,dpi−1,j,dpi−1,j−1+[ai==bj])dp_{i,j} = max(dp_{i,j-1} , dp_{i-1,j},dp_{i-1,j-1} + [a_i==b_j]) dpi,j​=max(dpi,j−1​,dpi−1,j​,dpi−1,j−1​+[ai​==bj​])

考虑到 KKK 比较小，对于每个 iii 只要记录其前 KKK 个的状态和后 KKK 个的状态就可以了。

Attention!

由于记录的是差分，而且这道题的差分数组不是从 000 开始的，为了知道真实值，需要记录一个真实的 dp\mathrm{dp}dp 值。dp\mathrm{dp}dp 要新开一维吗？不用。因为我们本身就要枚举前 iii 位失配了多少个位置，这恰恰是 i−dpi,i\mathrm{i-dp_{i,i}}i−dpi,i​ 。

J - Remote Control

GYM102759J

by jasonfan

题意

题目讲得挺清楚的。QAQ（我懒）

题解

考虑若不撞墙，可以直接算出终点，若知道在操作序列的第几次撞墙，那么终点也唯一确定了。

考虑预处理这些，需要知道 某个位置出发会在第几次操作撞墙、在某次操作撞墙后终点是啥。

前者简单，操作序列反向（L 变 R,U 变 D …），走到的位置若没标记则记上当前操作位置。

对于后者倒着处理，假设已经处理好 iii 之后每次操作撞墙后的终点，并维护了只管 iii 之后的操作序列，那些位置会在操作序列第几次撞墙。不断迭代计算就可以了。

I. Query On A Tree 17

题意

给你一颗大小为 NNN 的树，以 111 为根，带点权，最初所有点点权为 000，有 QQQ 次操作：

  • 1 u 给 uuu 子树内所有点权值加 111；
  • 1 u v 给 uuu 到 vvv 最短路径上所有点权值加 111。

操作是累加的，每次操作后输出树的带权重心（若有多个输出离根最近的）

2≤N≤105,1≤Q≤1052\le N \le 10^5,1\le Q\le 10^5 2≤N≤105,1≤Q≤105

题解

树的带权重心：从根出发，每次跳到权值和 ≥S2\ge\cfrac{S}{2}≥2S​ 的儿子，直到没有这样的儿子为止。

序列的带权中位数：从数列左端出发，如果左边（包含自己）的和 ≥S2\ge \cfrac{S}{2}≥2S​ 就往右跳。

考虑带权重心与将树的 dfn\mathrm{dfn}dfn 的带权中位数有什么关系。

可以发现 dfn\mathrm{dfn}dfn 序上的带权中位数对应的树上位置一定在树的带权重心的子树内，用反证法不难证明。

二分找到带权中位数后树上倍增即可。

C. Economic One-way Roads

GYM102759C

题意

给你 NNN 个点，需要你加边使其变成强连通图，加边有限制，对于两点 x,y(x<y)x,y(x<y)x,y(x<y)：

  • 告诉你 x,yx,yx,y 直接不允许连边
  • x→yx\to yx→y 代价为 ax,ya_{x,y}ax,y​，y→xy\to xy→x 代价 ay,xa_{y,x}ay,x​，你必须只能连一条边，且花费相应的代价。

问你最小代价，或判无解。N≤18N\le 18N≤18

题解

考虑状压 dp\mathrm{dp}dp，每次转移时，在已经属于强连通分量的点集中选两个点，然后向状态内添加由不属于强连通分量的点集的一条链。每次转移复杂度为 O(∣S∣22n−∣S∣)O(|S|^22^{n-|S|})O(∣S∣22n−∣S∣)。

但是这有问题，因为如果 u→vu \rightarrow vu→v 已经连通，再添加一条从 u→vu\rightarrow vu→v 的链肯定不是最优方案，只需保留一条。

最优方案的性质：由多个环拼起来，然后有的环会有一个或多个公用点，但不会出现单独的公共边。

如果出现一条单独的公共边，则形成一个更大的环。

基于这些性质，考虑 dpdpdp 状态，dp[S][u][v]\mathrm{dp[S][u][v]}dp[S][u][v] 表示已有的点集为 SSS，uuu 为入点，vvv 为出点。

转移的分类：

  • 入点的转移（零入点 →\rightarrow→ 有入点）
  • 出点的转移（零出点 →\rightarrow→ 有出点）
  • 连接入点和出点，变成零入点、零出点

以上是我口糊的做法，发现它很难实现，于是乎看了看题解做法：

事实上我们不需要知道两个入点和出点，我们只需要知道其中之一、以及它目标要到达的点（目标点）。

转移时只需要出点/入点转移就可以了。然后额外记录一个 F(S)\mathrm{F(S)}F(S) 表示在该状态下的最优强连通图。

转移一个状态之前用 F(S)\mathrm{F(S)}F(S) 更新所有点（注意是去 min⁡\minmin，而不是赋值）。

还有当出点与目标点相同时需要特判（不能出去后直接进来）。

B. Cactus Competition

题意

给你一个长度为 NNN 的序列 {Ai}\left\{A_i\right\}{Ai​} 和一个长度为 MMM 的序列 {Bi}\left\{B_i\right\}{Bi​}，表示一个 N×MN\times MN×M 的网格图，其中 (i,j)(i,j)(i,j) 格子的权值为 Ai+BjA_i+B_jAi​+Bj​。一个人从 (S,1)(S,1)(S,1) 出发到 (T,M)(T,M)(T,M)，每次只能从 (i,j)(i,j)(i,j) 到 (i+1,j)(i+1,j)(i+1,j) 或 (i,j+1)(i,j+1)(i,j+1) ，且不能经过 Ai+Bj<0A_i+B_j<0Ai​+Bj​<0 的格子。问有多少对可行的 S,TS,TS,T。

1≤N,M≤2×105,−109≤Ai,Bi≤1091\le N,M\le 2\times 10^5,-10^9\le A_i,B_i\le 10^9 1≤N,M≤2×105,−109≤Ai​,Bi​≤109

题解

  • ai≥ai−1a_i \ge a_{i-1}ai​≥ai−1​ ，第 iii 行可以经过的格子包含所有第 i−1i-1i−1 行可以经过的格子。
  • ai≤ai−1a_i \le a_{i-1}ai​≤ai−1​ ，第 iii 行可以经过的格子被第 i−1i-1i−1 行可以经过的格子包含。

由此可以得到一个结论：至少有一行的格子全部是可以通过的，否则方案数为 000。

可以发现，从 (1,1)(1,1)(1,1) 到 (n,m)(n,m)(n,m) 若不能到达，分为 444 中情况：

  • 行阻断，有一行全部 ≤0\le 0≤0，等价于 max⁡Ai+min⁡Bj<0\max_{A_i} + \min_{B_j} < 0maxAi​​+minBj​​<0。
  • 列阻断，有一列全部 ≤0\le 0≤0，等价于 min⁡Ai+max⁡Bj<0\min_{A_i} + \max_{B_j} < 0minAi​​+maxBj​​<0。
  • 起点阻断，起点被 <0<0<0 的格子匡在一个左上角，形式化地，∃(i,j),  Ai+By<0 (y<j)∧Ax+Bj<0 (x<i)\exist (i,j),\;A_i+B_y<0\,(y<j)\wedge A_x+B_j < 0\,(x<i)∃(i,j),Ai​+By​<0(y<j)∧Ax​+Bj​<0(x<i) 。
  • 终点阻断，终点被 <0<0<0 的格子匡在一个右下角，形式化地，∃(i,j),  Ai+By<0 (y>j)∧Ax+Bj<0 (x>i)\exist (i,j),\;A_i+B_y<0\,(y>j)\wedge A_x+B_j< 0\,(x>i)∃(i,j),Ai​+By​<0(y>j)∧Ax​+Bj​<0(x>i) 。

可以证明，如果不出现以上四种情况，则可以到达。

具体实现：

第一个条件，相当于把整个矩阵分成若干个不相干小矩阵，合法的点对不能跨越小矩阵。

第二个条件，只需要考虑 min⁡Bj\min_{B_j}minBj​​ 的那一列。这一列会出现一段段 <0< 0<0 的段，合法点对不能映射在同一段中。

第三个条件，将 AAA 降序排序后，双指针+倍增即可。

L - Steel Slicing 2

GYM102759L

by jasonfan

题意

给你一个长度为 nnn 的序列 {hi}\{h_i\}{hi​} 和 {li}\{l_i\}{li​} 表示上图所示的栅栏（所有边都是水平或竖直的），切一刀从某边界点出发水平或竖直知道边界点结束。问至少几刀使原图只留下矩形。

题解

其实就是给你一个凹多边形，让你把它变成凸多边形（只有水平、竖直边的多边形，若是凸的必然是矩形）。称向内凹的点为凹点，向外凸的点为凸点。一次切割只会改变切割边界点的性质，显然不会切割凸点，若割一个凹点会将其变成一个凸点+一个平面，若割到平面会将其变成两个凸点。当所有凹点都被删完自然就结束了。

考虑一次切割至少删除一凹点，若要删除两个凹点这两凹点必须在同一水平、竖直线上且之间没有经过边界。需要最大化删除两个凹点的操作次数。这些可能的操作很容易预处理，竖直的枚举一遍，水平的用单调栈。

并不是所有操作都可以随便选，如下图所示，红色的水平操作与蓝色的竖直操作向交，执行完一个，另一个就不行了。故给每个操作建点，相交的操作连边，要求最大独立集。考虑该图是二分图（水平操作互相不交，竖直操作互相不交），二分图中最大独立集等于总点数减最大匹配。

跑网络流？不需要，考虑一个 lll 到 rrr 的水平操作只会与 [l,r][l,r][l,r] 内的竖直操作有交，可以把竖直操作看成点，水平操作为区间，点匹配区间，贪心即可。
