XJ小清新DP

发表于2021-05-14|更新于2021-05-14|DP
|阅读量:

最近XJ几场NOI模拟赛自闭赛的小清新大毒瘤DP，赛时都没有做出来。都是质量挺高的DP题，刚好也覆盖了DP的三个方面：最值、计数、期望概率、数据结构优化，就整理在了一起方便复习。难度大致升序。

A.约定

题意

这一层有 nnn 个房间，，左数第 iii个房间在坐标轴的 aia_iai​ 位置。每个房间里都有一座钟，花费 d×⌊d⌋d\times \left\lfloor\sqrt{d}\right\rfloord×⌊d​⌋ 的体力敲钟，即可让左右距离 ddd
不超过 的房间都听到钟声（假定花费足够的体力就可以让任意远的房间都听到钟声）。

现在有一个有趣的问题：如果每个房间里都有一个人，现在某个时刻第 xxx 个房间里的人突然得到了一个信息，想要通过敲钟的方式，经过不超过 yyy 个时刻便通知到所有房间里的人。钟声传递需要一单位时间（不论距离多远）。如果大家事先安排好的话，就可以在完成上述任务的前提下最小化所有人花费的体力总和。注意这里有些人可以不选择敲钟，这样不花费任何体力。

现在星空希望你帮他求出，对所有的 (x,y),x∈[1,n],y∈[1,n)(x,y),x\in[1,n],y\in[1,n)(x,y),x∈[1,n],y∈[1,n)，最小可能花费的体力总和。

2≤n≤400,1≤a1<a2<⋯<an≤1092 \le n \le 400,1\le a_1 < a_2 < \dots < a_n \le 10^9 2≤n≤400,1≤a1​<a2​<⋯<an​≤109

题解

需要先得出一个结论：最优方案一定是从起点开始，向左右一个方向敲钟，然后扩展到某个位置，我们称之为最优中心，然后敲一个比较大的范围的，再左右各自扩展到 111 和 nnn。

于是我们可以先预处理出来从一个点向左右扩展到 111 和 nnn 的最小代价，就是先枚举第一次敲钟的范围，然后 DP 走到 111 和 nnn。

我们现在得到的 DP 数组相当于是花 0 时间在走到最优中心。然后我们枚举花多少时间走到最优中心去更新 DP 值。最后就得到了答案。

时间复杂度： O(n3)O(n^3)O(n3)

B.染色

题意

现在有个 n×mn \times mn×m 的棋盘，每次你可以将一行或者一个斜对角线从左下到右上）的格子给染黑，请问共有多少种可能的最终状态。

由于答案可能很大，答案对 modmodmod 取模。

108≤mod≤109+7n,m≤50010^8 \le mod \le 10^9 + 7\\ n, m \le 500 108≤mod≤109+7n,m≤500

题解

如果两种操作同时进行，会出现不必要的多余操作，导致算重。

所以，考虑先染斜对角，再染行。然而，我考场上想的是先染行，再染斜对角，所以 DP 式子非常的shit，没写出来。

然后每个斜对角所涉及的行可以表示为一个区间 [li,ri][l_i,r_i][li​,ri​]。然后假设所有的斜对角一开始都染了色，接下来就是枚举哪些斜对角没有染色。由于要避免算重，不能把一个斜对角的区间的所有元素都放满。

于是设计出 DP 状态：f(i,j)f(i,j)f(i,j) 表示到第 iii 个区间没有染色，当前区间最右边的 jjj 个选被中。

转移：枚举上一次没有染色的区间，可以染色的点 ∈∪[li,ri]\in \cup [l_i,r_i]∈∪[li​,ri​]。注意转移前面都染色的情况！
